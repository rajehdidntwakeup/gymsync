Critical: Authentication & Security

Make login work end-to-end and stop leaking passwords
Goal: The entire auth system is non-functional — login returns a placeholder string, no JWT filter exists, and password hashes leak in API responses. This phase fixes all of that.
1.1

Create JWT Utility Class

NEW security/JwtUtil.java

Create a utility class that handles JWT token creation and validation using the already-declared jjwt dependency.

  • generateToken(UserDetails) — uses gymsync.jwt.secret and gymsync.jwt.expiration from application.properties
  • extractUsername(String token) — extracts the subject claim
  • extractUserId(String token) — extracts the custom userId claim
  • isTokenValid(String token, UserDetails) — checks username match and expiration
  • extractAllClaims(String token) — private helper using Jwts.parserBuilder()
1.2

Make User Implement UserDetails + @JsonIgnore Password

FIX model/User.java

Add implements UserDetails to the User entity. This is needed for Spring Security authentication.

BEFORE
public class User {
    // ...
    @NotBlank @Size(max=120)
    private String password;

    public String getPassword() { return password; }
AFTER
public class User implements UserDetails {
    // ...
    @NotBlank @Size(max=120)
    private String password;

    @JsonIgnore
    public String getPassword() { return password; }

    public Collection<? extends GrantedAuthority> getAuthorities() {
        return Collections.singletonList(new SimpleGrantedAuthority("ROLE_USER"));
    }
    public boolean isAccountNonExpired() { return true; }
    public boolean isAccountNonLocked() { return true; }
    public boolean isCredentialsNonExpired() { return true; }
    public boolean isEnabled() { return true; }

Why: This fixes the critical password leak bug — without @JsonIgnore, the BCrypt hash is returned in every API response that includes a User object.

1.3

Create Custom UserDetailsService

NEW security/CustomUserDetailsService.java

Spring Security needs a way to load users from the database. This service bridges our UserRepository and Spring Security's UserDetailsService interface.

@Service
public class CustomUserDetailsService implements UserDetailsService {
    @Autowired
    private UserRepository userRepository;

    @Override
    public UserDetails loadUserByUsername(String username) {
        return userRepository.findByUsername(username)
            .orElseThrow(() -> new UsernameNotFoundException("User not found: " + username));
    }
}
1.4

Create JWT Authentication Filter

NEW security/JwtAuthenticationFilter.java

Create a filter that runs before Spring Security's default filter. It extracts the JWT from the Authorization: Bearer ... header, validates it, and sets the security context.

  • If header is present and valid → set SecurityContextHolder authentication
  • If header is missing → continue (allows permitAll endpoints through)
  • If token is expired/malformed → clear context, continue (secured endpoints will return 401)

1.6

Implement Login Endpoint (Replace Stub)

FIX controller/AuthController.java

Replace the placeholder string with real JWT token generation.

BEFORE
@PostMapping("/login")
public ResponseEntity<String> login(@RequestBody LoginRequest request) {
    // TODO: Implement JWT token generation
    return ResponseEntity.ok("Login endpoint - JWT implementation pending");
}
AFTER
@PostMapping("/login")
public ResponseEntity<?> login(@RequestBody LoginRequest request) {
    Authentication authentication = authenticationManager.authenticate(
        new UsernamePasswordAuthenticationToken(
            request.getUsername(), request.getPassword()
        )
    );
    UserDetails userDetails = (UserDetails) authentication.getPrincipal();
    String token = jwtUtil.generateToken(userDetails);
    User user = userRepository.findByUsername(userDetails.getUsername())
        .orElseThrow(() -> new RuntimeException("User not found"));
    return ResponseEntity.ok(Map.of(
        "token", token, "type", "Bearer",
        "userId", user.getId(), "username", user.getUsername()
    ));
}

Also: inject AuthenticationManager and JwtUtil, and make LoginRequest public.

Fix Mobile Auth Flow

FIX AuthContext.tsx (mobile + mobile-expo)

BEFORE
const login = async (credentials) => {

AFTER

    if (token) {
        const response = await api.get('/users/me');
        setUser(response.data);
    }
};
1.9

Fix 401 Interceptor Navigation

FIX api.ts (mobile + mobile-expo)   MODIFY App.tsx

When the 401 interceptor fires, it clears the token but doesn't redirect to login. Add a callback mechanism:

// In api.ts:
let onUnauthorizedCallback: (() => void) | null = null;
export const setOnUnauthorized = (cb: () => void) => { onUnauthorizedCallback = cb; };

// In the 401 handler:
if (error.response?.status === 401) {
    await AsyncStorage.removeItem('token');
    onUnauthorizedCallback?.();  // triggers navigation reset
}

// In App.tsx — register the callback:
React.useEffect(() => {
    setOnUnauthorized(() => {
        navigationRef.reset({ index: 0, routes: [{ name: 'Login' }] });
    });
}, []);

Fix ChatScreen WebSocket Connection Leak

FIX ChatScreen.tsx (mobile + mobile-expo)

Switching chat partners creates a new WebSocket connection without closing the previous one, leading to connection leaks and duplicate message handlers.

BEFORE
useEffect(() => {
  if (selectedPartner) {
    connectSocket();
    loadHistory();
  }
}, [selectedPartner]);
AFTER
useEffect(() => {
  if (selectedPartner) {
    chatSocket.disconnect();  // close previous connection
    connectSocket();
    loadHistory();
  }
  return () => { chatSocket.disconnect(); };
}, [selectedPartner]);
